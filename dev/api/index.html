API

DualSignals.Component — Type

Component in the model (node, link, sector, etc.).

component_id should be unique within a dataset.

source
DualSignals.ComponentType — Type

High-level category for model components.

source
DualSignals.Constraint — Type

Constraint metadata with sense, kind, and component references.

component_ids should refer to Component.component_id values.

source
DualSignals.ConstraintKind — Type

High-level category for constraints.

  • balance: flow or nodal balance.
  • capacity: upper/lower bounds or limits.
  • resource: resource availability constraints.
  • policy_cap: policy-driven caps (emissions, quotas).
  • technology: technology-driven bounds.
source
DualSignals.ConstraintSense — Type

Constraint sense (<=, =, >=).

  • le: less-than-or-equal
  • eq: equality
  • ge: greater-than-or-equal
source
DualSignals.ConstraintSolution — Type

Solution record for a constraint, including dual and optional slack/activity.

dual is required; activity, slack, and is_binding are optional.

source
DualSignals.DatasetMetadata — Type

Metadata describing the dataset and objective context.

Fields:

  • description: optional description of the dataset.
  • created_at: optional timestamp for when the dataset was generated.
  • objective_sense: minimize or maximize if known.
  • objective_value: objective value at the solution if known.
  • units_convention: optional units description (e.g., "MW, USD/MWh").
  • notes: free-form notes for provenance and assumptions.
source
DualSignals.DualSignalsDataset — Type

Top-level dataset containing metadata, components, constraints, and solutions.

dataset_id should be unique for storage or exchange.

source
DualSignals.ObjectiveSense — Type

Objective sense for interpreting dual impacts.

  • minimize: duals are interpreted directly.
  • maximize: duals are interpreted with reversed sign.
source
DualSignals.VariableValue — Type

Value of a variable associated with a component.

name identifies the variable (e.g., "flow", "production", "value").

source
DualSignals.aggregate_duals — Method

Aggregate dual values by constraint kind, id, or component id.

Keywords:

  • by: :kind, :constraint_id, or :component_id.
  • scenario: optional scenario filter.
  • time: optional time filter.

Returns a vector of named tuples with key, count, mean, mean_abs, and max_abs.

source
DualSignals.aggregate_duals_series — Method

Aggregate dual values over time or scenario for each key.

Keywords:

  • by: :constraint_id, :kind, or :component_id.
  • over: :time or :scenario.
  • scenario: optional scenario filter.

Returns a vector of named tuples with key, over, count, mean, mean_abs, and max_abs.

source
DualSignals.bindingness — Method

Return per-constraint rows with slack, binding flag, and impact labels.

Keywords:

  • tol: tolerance for binding status when slack is inferred.

Returns a vector of named tuples with: constraint_id, kind, sense, rhs, activity, slack, dual, is_binding, impact, time, scenario.

source
DualSignals.isvalid_dataset — Method

Return true if the dataset has no validation errors.

source
DualSignals.jump_dataset — Method

Extract a DualSignalsDataset from a JuMP model and its solution.

Keywords:

  • dataset_id: dataset identifier for the exported results.
  • include_variables: include variable values.
  • include_constraints: include constraint metadata.
  • include_constraint_solutions: include constraint solutions and duals.
  • units_convention: units description for metadata.
  • description: optional dataset description.
  • kind_hint: function mapping constraint label -> ConstraintKind.
  • tag_hint: function mapping (label, kind) -> tags vector or nothing.
source
DualSignals.narrative_policy_summary — Method

Generate a narrative policy summary with optional extras.

Keywords:

  • top: number of rows per summary section.
  • tol: tolerance for binding status.
  • slack_change: delta slack for relaxation-value metric.
  • include_duration: include time-weighted bottlenecks (:auto or Bool).
  • include_slack_change: include relaxation value (:auto or Bool).
source
DualSignals.narrative_top_bottlenecks — Method

Generate a short narrative summary of top bottlenecks.

Keywords:

  • top: number of bottlenecks to include.
  • tol: tolerance for binding status.
source
DualSignals.rank_constraints — Method

Rank constraints by dual-based metrics and return the top rows.

Keywords:

  • metric: :abs_dual, :dual, :dual_times_slack, :dual_times_binding_duration, or :dual_times_slack_change.
  • top: number of rows to return.
  • kind: filter by ConstraintKind.
  • scenario: filter by scenario value.
  • time: filter by time value.
  • tol: tolerance for binding status.
  • binding_only: if true, return only binding constraints.
  • slack_change: value for :dual_times_slack_change metric.

Returns a vector of named tuples with ranking details.

source

DualSignals.table_policy_priorities — Method

Return a bundle of tables for bottlenecks and capacity priorities.

  • with_impact: include human-readable impact labels.
  • slack_change: delta slack for relaxation-value metric.

  • include_slack_change: include relaxation value (:auto or Bool).
source

  • top: number of rows to return.
  • tol: tolerance for binding status.
  • binding_only: if true, return only binding constraints.
source
